# Landscape skill framing

Progressive disclosure for **naming** skill theses. Not a taxonomy — prompts that
travel outside one employer. Domain mix shifts; the naming rules do not.

## Naming rule

**Title = skill a stranger recognizes.** Evidence = your MRs and optional shelves.

| Weak (epic / ticket) | Stronger (skill thesis) |
| --- | --- |
| “Phase 2 NVMe swap” | “Isolated code-exec at real capacity” |
| “Port Viewer to Next” | “Ship a production cloud host from desktop DNA” |
| “Add share links” | “Layered access modes with URI trust boundaries” |

## Merge rule

If two work programs are one user-visible skill (e.g. artifact UX + runtime limits
→ “operable agent side effects”), **one substrate study**.

## Defer rule

Docs taxonomies, CI job renames, and pure cleanup usually go to `deferred/` unless
they uniquely prove measurement-gated rollout or teachable-systems craft the user
wants on the portfolio.

## Prompt themes (refresh as hiring landscape moves)

Use as ignition, not a checklist. Drop rows that don't match the evidence; add
rows the evidence demands.

| Skill thesis pattern | What evidence often looks like |
| --- | --- |
| Production MCP / agent-tool infrastructure | Servers, Resources vs tools, auth, skill packs, incomplete-client adapters |
| Operable agent tool loops | Backpressure, cancel/interrupt, typed errors/death, recovery |
| Durable side effects from agent runs | Artifacts, object-storage barriers, product-consumable URIs |
| Isolated code-exec / sandbox compute | MicroVMs, multi-tenant isolation, sizing, validation gates |
| Vertical / domain-grounded agents | Runtime fitness, domain skills, environment overlays |
| Multi-surface product architecture | Host birth → shared extraction → dual-host converge |
| Host vs shared ownership | Dependency-inverted ports; hosts keep persistence/I/O engines |
| Privilege-bounded native capability | Capability allowlists, IPC trust, credential placement |
| Versioned domain load paths | Schema/version gates across hosts |
| 0→1 platform with kill-switches | Scaffold → integrate → converge → delete transitional path |
| Measurement-gated rollout | Harnesses that can FAIL the merge bar |
| Teachable systems (optional meta) | Glossaries, teaching spines, thin ADRs — only if user wants this piece |
