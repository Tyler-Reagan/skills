# Case study substrate template

Dense **substrate** for one **skill thesis**. Not an interview highlight card —
no Owned/Interfaced fences, no suite budget, no collapse hierarchy.

```markdown
# {Skill thesis as title — not epic name}

**Role:** …
**Timeline:** …
**Repos(s) / surface:** …

**Summary:** {One or two sentences: skill thesis + what shipped. Qualitative impact only.}

## Problem
{Technical + product constraint. What made the naive path fail?}

## Spine
{Exemplar MR/issue sequence — ordered proof, not a backlog dump.}
1. …
2. …

## Decisions
{2–5 judgments + trade-offs. Cite ADRs when real.}
- …

## What shipped
{Concrete surfaces, kill-switches, migrations. Evidence handles.}

## Stack
{Named tech that constrained the work — depth over buzzwords.}

## Agency
{What you personally drove or decided.}

## Outcome
{Honest qualitative results. Real metrics only if they exist — never invent.}

## Shelves (optional)
{Taught pages, ADRs, design docs that support claims — links only.}

## Deferred / out of fence
{Related work parked or owned by a sibling arc — name it so packaging doesn't re-absorb it.}
```

## Density bar

A study is thick enough when a later highlights pass can form **Decisions** and a
**Spine** without re-mining git. If packaging would need another history search,
the substrate is thin — expand here, not in cards.
