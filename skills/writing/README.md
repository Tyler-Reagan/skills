# Writing Skills

Doc-type formatting and the personal portfolio pipeline — turning code, git history, and existing docs into standards-conformant writing.

## Skills

| Skill | Role | Use when |
|---|---|---|
| [`doc-formatting`](doc-formatting/SKILL.md) | Formatting | Formatting a doc to its type's contract — architecture (state-of-code, diagram-led), decision (transcript-of-truth record), plan (active work doc) |
| [`portfolio-arcs-from-history`](portfolio-arcs-from-history/SKILL.md) | Specialist | Mining live repos and git/GitLab history into personal portfolio substrate — skill theses, dense case studies, a skill-map README |
| [`portfolio-arcs-maintain`](portfolio-arcs-maintain/SKILL.md) | Specialist | Keeping portfolio arcs current as new work lands — update-in-place by default, new substrate/highlights only past the novelty bar |
| [`portfolio-arcs-public-nuggets`](portfolio-arcs-public-nuggets/SKILL.md) | Specialist | Distilling internal portfolio arcs into public-safe, recruiter-facing nuggets |
| [`portfolio-swe-highlights`](portfolio-swe-highlights/SKILL.md) | Specialist | Packaging portfolio substrate into SWE interview highlight cards under a hard budget |

> [!NOTE]
> The `portfolio-*` skills set `disable-model-invocation: true` — they're personal-use-only and must be invoked explicitly by name rather than triggered by description matching.
